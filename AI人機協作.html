01Design Philosophy

和「一鍵生成」相反：
我們刻意讓 AI 不替你做完

外面很多 AI 工具強調「一鍵生成」，輸入問題、直接吐答案。這套課程設計的 Agent 邏輯不同——強調的是不代勞、要引導。AI 負責幫你思考、幫你整理，但最後那個判斷與決策，刻意留給人來做。

講者舉了聯發科的例子：AI 可以替主管省下大量寫報告、查資料的時間，但最後的決策、那個心智模式的判斷，還是得由人來下。這正是「人才是核心」的具體展現。

關鍵心法

AI 是「來幫助你思考」的，不是「來取代你思考」的。把流程裡可重複的部分交給它，把判斷與決策留給自己——這才是健康的協作關係。

AI 不僅僅是一個工具，它更像是一個可以跟你協作的夥伴——不罷工、不休息，隨時隨地陪伴你。

02Three Agents

把「聽得懂、回去卻不會用」的
管理工具，變成可操作系統

看問題五層次、OGSM、Ideas、績效面談——這些管理工具上課時都聽得懂，回到崗位卻不知道怎麼用。今天的做法，是透過 AI Agent 把它們變成可執行、可操作的輔助系統。當天交付的，是三套指令集：

01

看問題五層次 Agent

把「拆解問題」的思考框架做成可對話的引導流程，帶使用者一層層往下追到根因，而不是直接給結論。

02

Ideas Agent

把發想與規劃的方法論轉成可操作的輔助系統，協助使用者把模糊的想法結構化。

03

面談軍師 Agent

把績效面談這類最需要臨場判斷、最難標準化的場景，做成事前可演練、事中可參考的軍師。

由 兔子 分享
這三套指令集代表什麼

它們是一整天課程的核心濃縮：AI 是加速器，是把原本要做很久、想很久、甚至想不清楚的事，變得更有條理、更快速的工具。

03Tool → Agent → AI

你也能成為團隊的 AI 教練，
刻出屬於你們部門的 Agent

講者給主管的建議：別把這些工具當成新奇玩具。回去之後，在團隊裡推動「AI 協作」的工作模式——而且不是由上而下強迫大家用，而是你自己先用，用了確實有幫助，再分享給團隊。

Step 01
教工具
學會怎麼操作 AI 工具，先把基本功用起來。
Step 02
教 Agent
把專業知識與 SOP 寫進指令集，刻出能引導他人的 Agent。
Step 03
建 AI
把 Agent 系統化，變成企業內部的智慧資產。

把專業知識、把 SOP 寫進指令集，等於擁有無數個不知疲倦的超級助理。每個部門都能依自己的需求刻出 Agent：

04Mindset

面對會顛覆工作模式的新科技，
別停在「抗拒」這一步

很多時候，面對像 AI 這種看起來會顛覆工作模式的東西，第一反應都是抗拒，或覺得「這只是一時的時髦」。但這個時代變動得太快了，不能再用過去的思維去面對未來的挑戰。

第一反應
抗拒 · 覺得只是一時時髦
→
該有的心態
為了生存與競爭力而擁抱
擁抱 AI，不是為了追求時髦，而是為了生存——為了在競爭激烈的市場裡，保持我們的競爭力。

Field Notes · 筆記者綜合觀察

整場收尾，其實只在反覆說一件事：
AI 放大你，但生不出那個「你」。

三套 Agent、三層階梯（教工具 → 教 Agent → 建 AI）、一個心態轉折——表面講的是工具，底層講的都是同一件事：先把你的專業整理成 AI 能讀懂的結構與指令，AI 才能替你加速。它能把「一」放大成「十」，卻無法替你生出那個「一」。

協作觀

主張不代勞、要引導，刻意把決策留給人。
理由AI 省下查找與整理的時間，心智判斷仍需人下。
啟示把可重複的交給它，把判斷握在自己手裡。

推動觀

主張由下而上：自己先用，有效再分享。
理由強迫使用難以生根，親身受益才會擴散。
啟示當教練，用指令集把 SOP 變成部門 Agent。
當工具做好了，工具是來協助你思考的。
該把時間，從操作工具，還給決策。
